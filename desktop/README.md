# Atomic Bot Desktop

Cross-platform Electron desktop app for [Atomic Bot](https://atomicbot.ai) — an AI assistant that makes things for you.

**Platforms:** macOS (arm64 / x64) · Windows (x64)

## Quick Start

```bash
# Install dependencies (from repo root)
pnpm install

# Prepare bundled runtimes
npm run prepare:all

# Build & launch in dev mode
npm run dev
```

## Architecture

The app follows a standard Electron multi-process model with a clear separation of concerns:

```
apps/electron-desktop/
├── src/                    # Main process + preload
│   ├── main.ts             # Electron entry point
│   ├── preload.ts          # Context bridge (window.openclawDesktop)
│   ├── shared/             # IPC channel definitions & bridge contract
│   └── main/
│       ├── bootstrap/      # App wiring & lifecycle
│       ├── gateway/        # OpenClaw gateway spawn & config
│       ├── ipc/            # IPC handlers (one module per domain)
│       ├── platform/       # OS abstraction (darwin / win32)
│       ├── whisper/        # Speech-to-text (Whisper)
│       ├── gog/            # Gog runtime integration
│       ├── terminal/       # PTY manager (multi-session)
│       ├── window/         # Window & tray management
│       └── keys/           # API key storage & auth profiles
├── renderer/               # Renderer process (Vite + React)
│   └── src/
│       ├── main.tsx        # React root (Redux Provider, HashRouter)
│       ├── gateway/        # WebSocket RPC client & context
│       ├── store/          # Redux Toolkit slices
│       └── ui/             # Feature-based UI modules
│           ├── chat/       # Chat interface & message rendering
│           ├── settings/   # Settings (connectors, providers, skills)
│           ├── onboarding/ # First-run setup wizard
│           ├── terminal/   # Embedded terminal
│           ├── sidebar/    # Session navigation
│           └── shared/     # Reusable UI kit & utilities
├── assets/                 # Icons, tray images, error/loading HTML
├── scripts/                # Build & bundle scripts
├── e2e/                    # Playwright E2E tests
└── vendor/                 # Bundled runtimes (generated by prepare scripts)
```

### Process Communication

All IPC flows through a typed contract defined in `src/shared/`:

```
Renderer  →  preload.ts (contextBridge)  →  Main process (ipcMain.handle)
          ←  IPC_EVENTS (main→renderer)  ←
```

Channel names live in `ipc-channels.ts`; the full API surface is declared in `desktop-bridge-contract.ts`.

## Scripts

### Development

| Command                  | Description                                           |
| ------------------------ | ----------------------------------------------------- |
| `npm run dev`            | Rebuild native modules, build all, launch Electron    |
| `npm run build`          | Compile main process (tsc) + bundle preload (esbuild) |
| `npm run build:renderer` | Build renderer with Vite                              |
| `npm run build:all`      | Build main + renderer                                 |

### Runtime Preparation

| Command                          | Description                                             |
| -------------------------------- | ------------------------------------------------------- |
| `npm run prepare:all`            | Bundle OpenClaw core + all runtimes                     |
| `npm run prepare:openclaw`       | Bundle OpenClaw into `vendor/openclaw`                  |
| `npm run prepare:runtimes`       | Fetch & prepare Node, Gog, jq, memo, and other runtimes |
| `npm run verify:openclaw-bundle` | Verify the bundled OpenClaw is loadable                 |

### Testing

| Command                  | Description                                 |
| ------------------------ | ------------------------------------------- |
| `npm run test`           | Run unit tests (Vitest)                     |
| `npm run test:watch`     | Run unit tests in watch mode                |
| `npm run test:coverage`  | Run unit tests with coverage                |
| `npm run test:e2e`       | Prepare bundle + build + run Playwright E2E |
| `npm run test:e2e:quick` | Build + run E2E (skip bundle prep)          |

| Variable                      | Context  | Description                                             |
| ----------------------------- | -------- | ------------------------------------------------------- |
| `VITE_BACKEND_URL`            | Renderer | Override API backend URL (set in `renderer/.env.local`) |
| `OPENCLAW_DESKTOP_NODE_BIN`   | Main     | Custom Node binary path for development                 |
| `CSC_IDENTITY_AUTO_DISCOVERY` | Build    | Set to `false` to skip code signing                     |

## Adding New Features

### New IPC Channel

1. Add channel name to `src/shared/ipc-channels.ts`
2. Add method signature to `src/shared/desktop-bridge-contract.ts`
3. Implement bridge in `src/preload.ts`
4. Add handler in `src/main/ipc/<domain>-ipc.ts`
5. Register in `src/main/ipc/register.ts`

### New UI Feature

1. Create a directory under `renderer/src/ui/`
2. Register the route in `ui/app/routes.ts`
3. Shared components go in `ui/shared/kit/`

### New Bundled Binary

1. Add field to `BinaryPaths` in `src/main/types.ts`
2. Add `resolveBin()` call in `src/main/bootstrap/app-bootstrap.ts`
3. Add `extraResources` entry in `package.json` build config
